## 🤝 贡献指南

欢迎提交 PR 来丰富这份清单！请遵循以下格式：

### 表格字段说明

```
| [项目名称](仓库地址) | 一句话描述 (≤30字) | 核心技���栈 | ⭐ Stars | Demo链接 | 仓库链接 |
```

### 提交规范

1. **Fork** 本仓库，在新分支上修改
2. 将新项目添加到**对应技术生态**分类的表格中
3. **技术栈** 列请用 ` · `（空格+中点+空格）分隔
4. Stars 列使用 shields.io 徽章：`![Stars](https://img.shields.io/github/stars/用户名/仓库名?style=social)`
5. Demo 列无链接时填写 `—`
6. 提交前确认项目是**开源**且**仍在维护**的
7. Commit message: `Add: [项目名称]`

### 收录标准

- ✅ 开源（有 OSI 兼容协议）
- ✅ 功能完整的后台 / OA / 中后台系统
- ✅ 有一定使用文档
- ❌ 纯 UI 模板（无业务逻辑）
- ❌ 已停止维护超过 2 年
- ❌ 需要付费授权
